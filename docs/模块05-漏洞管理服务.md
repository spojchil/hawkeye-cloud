# 漏洞管理服务（vul-service）

**端口：** `:8004`
**状态：** 已完成

---

## 定位

管理漏洞检测模板（POC），定义"检测什么"和"怎么检测"。是核心检测链路的**知识库**。

## 子模块结构

| 子模块 | 关键类 | 职责 |
|--------|--------|------|
| vul-common | 12 个 Entity、枚举（VulSeverityEnum）、DTO/VO、配置 | 实体、枚举、VO |
| vul-api | `VulTemplateController`、`VulTemplateImportController`、`VulCategoryController`、`VulTagController` | REST API |
| vul-business | `VulTemplateServiceImpl`、`VulTemplateImportServiceImpl`、`VulCategoryServiceImpl`、`VulTagServiceImpl`、12 个 Mapper、3 个 MapStruct | 业务逻辑 |
| vul-bootstrap | `VulApplication` | 启动类 |

## 数据模型（12 张表）

| 表 | 说明 | 关键字段 |
|----|------|----------|
| `vul_template` | 模板主表 | template_id, name, severity, cve_id, cvss_score, flow, variables, enabled |
| `vul_text_content` | 文本内容 | description（Markdown） |
| `vul_http_step` | HTTP 探测步骤 | step_order, method, path, raw, headers, body, attack |
| `vul_matcher` | 匹配器配置 | type(Word/Status/Dsl/Regex), condition, negative, config |
| `vul_extractor` | 提取器配置 | type(Regex/Kval), name, config |
| `vul_reference` | 参考链接 | url, title |
| `vul_category` | 漏洞分类（树形） | name, parent_id |
| `vul_category_mapping` | 模板-分类关联 | category_id, template_id |
| `vul_tag` | 标签 | name（自动创建） |
| `vul_template_tag` | 模板-标签关联 | template_id, tag_id |
| `vul_template_category` | 模板-分类关联 | template_id, category_id |
| `detection_result` | 检测结果 | task_item_id, status, response_status_code, matched_at |

## API 端点

### 模板接口 (VulTemplateController)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/templates` | 分页查询（支持按名称/严重程度/标签/分类/启用状态过滤） |
| GET | `/templates/{id}` | 获取详情（含 httpSteps/matchers/extractors/references） |
| POST | `/templates` | 创建模板 |
| PUT | `/templates/{id}` | 更新模板（部分更新） |
| DELETE | `/templates/{id}` | 软删除模板 |
| PATCH | `/templates/{id}/enabled` | 快速启用/禁用 |

### 模板导入 (VulTemplateImportController)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/templates/import` | YAML 解析导入，级联入库（模板 + 步骤 + 匹配器 + 提取器 + 标签 + 分类） |

### 分类接口 (VulCategoryController)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/categories` | 查询分类树（含子节点 + 模板计数） |
| POST | `/categories` | 创建分类 |
| PUT | `/categories/{id}` | 更新分类（防环检查） |
| DELETE | `/categories/{id}` | 删除分类（有关联时拒绝） |
| POST | `/categories/{id}/templates` | 批量关联模板 |
| DELETE | `/categories/{id}/templates` | 批量取消关联 |
| GET | `/categories/{id}/templates` | 分类下模板列表（分页） |

### 标签接口 (VulTagController)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/tags` | 标签列表（含模板计数） |

### 内部 Feign 接口（供 task-service 调用）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/internal/templates/{id}` | 获取模板检测配置（精简版，仅含执行所需字段） |
| POST | `/internal/templates/batch` | 批量获取检测配置 |

## 模板导入

支持 YAML 格式（兼容 Nuclei 模板格式）批量导入：
- 自动解析 YAML → 级联创建 template + text_content + http_step + matcher + extractor + reference
- 标签和分类自动创建或复用已有
- 重复 template_id 时抛异常拒绝

## 设计要点

1. **知识库模式** — vul-service 只管存储模板，不执行检测
2. **软删除** — 使用 `deleted_at` 时间戳
3. **JSON 字段** — variables, config, path, headers 等以 JSON 存储
4. **Feign 精简** — 内部接口只返回检测执行必需的字段，不含 name/description/CVE 等元数据
