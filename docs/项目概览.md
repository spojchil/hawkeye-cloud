# Hawkeye Cloud — 项目概览

## 项目定位

Hawkeye Cloud 是一个**分布式漏洞检测平台**，采用微服务架构设计，支持**多租户 SaaS 部署**。目标是为企业用户提供可扩展、高可用的资产安全检测服务。

> 当前处于 **第二阶段（服务基础 CRUD 完成）**，网关、认证、资产三个基础服务已就绪，下一步将搭建核心检测链路：漏洞管理 → 任务调度 → 检测执行。

---

## 项目目标

- 构建一个完整、可落地的微服务项目
- 遵循企业级最佳实践：统一异常处理、多租户、分页、日志切面、代码规范

## 模块依赖关系

```
common-utils ◀─── auth-common ◀─── auth-business ◀─── auth-api ◀─── auth-bootstrap
common-utils ◀─── asset-common ◀─── asset-business ◀─── asset-api ◀─── asset-bootstrap
common-utils ◀─── vul-common ◀─── vul-business ◀─── vul-api ◀─── vul-bootstrap
common-utils ◀─── task-common ◀─── task-business ◀─── task-api ◀─── task-bootstrap
common-utils ◀─── detection-common ◀─── detection-business ◀─── detection-bootstrap
common-utils ◀─── tenant-common ◀─── tenant-business ◀─── tenant-api ◀─── tenant-bootstrap
common-utils ◀─── audit-common ◀─── audit-business ◀─── audit-api ◀─── audit-bootstrap
gateway-common ◀─── gateway-business ◀─── gateway-bootstrap
```

- `common-utils` 是所有业务模块的公共依赖
- 每个服务按 **common → business → api → bootstrap** 四层分包，职责清晰
- 启动器（bootstrap）只依赖 API 层，API 层依赖业务层，业务层依赖公共层
- **核心检测链路**：`task-service` 调用 `vul-service`（获取漏洞模板）和 `asset-service`（获取资产信息），拆分后通过 RocketMQ 分发给 `detection-service`

### 核心检测链路（四服务）

```
asset-service ──(资产信息)──┐
                            ├──► task-service ──(RocketMQ)──► detection-service
vul-service ───(漏洞模板)───┘         │                              │
                               预处理队列                        工作队列
                          (校验 + 拆分 + 分发)             (并发探测 + 负载上报)
                                                                │
                                                         失败重试耗尽
                                                                ▼
                                                           死信队列
                                                        (补偿 + 诊断)
```

## 通用能力

- **统一响应格式** — `ApiResponse<T>` + 业务异常 `ApiException` + 错误码 `ErrorCode`
- **统一分页模型** — `ListResult<T>` 封装分页返回
- **多租户支持** — 基于 MyBatis-Plus 的 `TenantLineHandler`，从请求头 `X-TENANT-ID` 自动注入
- **自动填充** — `MetaObjectHandler` 自动填充 `create_time`、`update_time`、`deleted`
- **日志切面** — `@LogExecutionTime` 注解 + AOP 实现方法执行耗时统计
- **MapStruct 映射** — 实体 ↔ DTO ↔ VO 对象转换
