# Detection-Service 架构设计 v3.0

> 纯 Worker 执行引擎。从 RocketMQ 消费消息，执行 HTTP 探测 + 匹配 + 提取 + DSL 判定，批量写结果。
> 不暴露 REST 接口，零外部 Feign 调用。

---

## 一、整体分层

```
┌────────────────────────────────────────────┐
│  TaskItemConsumer                          │  ← 入口：收消息、设租户上下文、调 Engine
├────────────────────────────────────────────┤
│  DetectionEngine                           │  ← 编排：解析 flow → 逐步骤执行 → 汇总结果
├────────────────────────────────────────────┤
│  MatcherPipeline   ─→ MatcherRegistry      │  ← 匹配：按 type 找策略 → and/or/negative 编排
│  ExtractorPipeline ─→ ExtractorRegistry    │  ← 提取：同上
│  FlowInterpreter                           │  ← 流程：顺序 / && / ||
│  HttpExecutor                              │  ← 协议：连接池、变量解析、http/raw 双模式
│  DslEngine (Aviator)                       │  ← DSL：自定义函数注册 + 表达式执行
│  VariableContext                           │  ← 变量：优先级分层 + {{...}} 解析
├────────────────────────────────────────────┤
│  ResultWriter                              │  ← 写入：ConcurrentLinkedQueue → saveBatch 批量
│  DetectionResultMapper                     │
└────────────────────────────────────────────┘
```

**核心原则：** 每种能力（匹配方式、提取方式）都是可插拔的 `@Component`。新增一种策略 = 写一个类，不改任何已有文件。

## 二、匹配器体系（Matcher）

### 2.1 策略接口

```java
public interface Matcher {
    /** 类型标识，对应 DB 中 vul_matcher.type */
    String type();

    /** 对 HTTP 响应执行匹配 */
    boolean match(HttpResponseContext ctx, MatcherDef def);
}
```

### 2.2 MatcherDef — 胖 DTO，编译期类型安全

```java
@Data
public class MatcherDef {
    private String type;              // word / status / dsl / regex / binary / size / xpath
    private String part;              // body / header / all / content_type / ...
    private String condition;         // inner and/or — 单 matcher 内多条规则的关系
    private boolean negative;
    private boolean caseInsensitive;
    // 以下按 type 使用，策略内部只读自己关心的字段
    private List<String> words;
    private List<Integer> status;
    private List<String> dsl;
    private List<String> regex;
    private List<String> binary;
    private List<Integer> size;
    private List<String> xpath;
}
```

**为什么不用 Map？** 字段就是实际存在的规则类型，编译期检查，IDE 补全。策略只读自己关心的字段，不写 `config.get("words")`。

### 2.3 注册表 — Spring 自动收集

```java
@Component
public class MatcherRegistry {
    private final Map<String, Matcher> map;

    public MatcherRegistry(List<Matcher> list) {
        this.map = list.stream().collect(toMap(Matcher::type, identity()));
    }

    public Matcher get(String type) { return map.get(type); }
}
```

新增匹配器：写一个 `@Component class XxxMatcher implements Matcher {}`，框架自动发现。

### 2.4 编排器

```java
@Component
public class MatcherPipeline {
    private final MatcherRegistry registry;

    public boolean evaluate(HttpResponseContext ctx, List<MatcherDef> defs, String outerCondition) {
        if (defs == null || defs.isEmpty()) return false;
        Predicate<MatcherDef> evaluator = def -> {
            Matcher m = registry.get(def.getType());
            boolean result = m.match(ctx, def);
            return def.isNegative() != result;
        };
        return "and".equals(outerCondition)
            ? defs.stream().allMatch(evaluator)
            : defs.stream().anyMatch(evaluator);
    }
}
```

### 2.5 实现清单

| 策略 | 类型 | 优先级 |
|------|------|--------|
| `WordMatcher` | word | P0 — 68% 模板 |

## 三、提取器体系（Extractor）

镜像 Matcher 设计。

```java
public interface Extractor {
    String type();                    // regex / kval / json / dsl / xpath
    String extract(HttpResponseContext ctx, ExtractorDef def);
}
```

```java
@Component
public class ExtractorRegistry {
    private final Map<String, Extractor> map;
    public ExtractorRegistry(List<Extractor> list) { ... }
    public Extractor get(String type) { return map.get(type); }
}
```

```java
@Component
public class ExtractorPipeline {
    private final ExtractorRegistry registry;

    public void extract(HttpResponseContext ctx, List<ExtractorDef> defs, VariableContext vars) {
        if (defs == null) return;
        for (ExtractorDef def : defs) {
            Extractor e = registry.get(def.getType());
            String value = e.extract(ctx, def);
            if (value != null && def.getName() != null) {
                vars.set(def.getName(), value);
            }
        }
    }
}
```

| 策略 | 类型 | 优先级 |

| `RegexExtractor` | regex | P0 — 16% 模板 |
| `KvalExtractor` | kval | P1 — 1.2% |
| `JsonExtractor` | json | P1 — 1.3% |
| `DslExtractor` | dsl | P1 — 1.8% |
| `XPathExtractor` | xpath | P2 — 0.2% |

## 四、DSL 引擎 — Aviator

### 4.1 为什么换掉 MVEL

| 维度 | MVEL 2.x | Aviator |
|------|----------|---------|
| 维护 | 2017 年停止 | 活跃（2025） |
| 安全 | 可执行任意 Java 代码，有已知 CVE | 沙箱，默认禁反射/IO |
| 性能 | 解​释​执​行 | 编译为字节码，5-10x |
| 语法 | `contains`/`regex` 是关键字，需要 preprocess 扭曲 | 直接当函数名用 |

### 4.2 用法

```java
@Component
public class DslEngine {
    public DslEngine() {
        AviatorEvaluator.addFunction(new ContainsFunction());
        AviatorEvaluator.addFunction(new RegexFunction());
        AviatorEvaluator.addFunction(new ToLowerFunction());
        AviatorEvaluator.addFunction(new CompareVersionsFunction());
        // ...
    }

    public boolean evaluate(String expression, Map<String, Object> env) {
        return (boolean) AviatorEvaluator.execute(expression, env);
    }
}
```

DSL 表达式 `contains(body, 'admin') && status_code == 200` 直接执行，不需要 `preprocess()`。

### 4.3 函数优先级

| 优先级 | 函数 | 出现次数 |
|--------|------|---------|
| P0 | regex, contains, contains_all, tolower | 67,000+ |
| P0 | compare_versions | 372 |
| P1 | contains_any, concat, len | 660+ |
| P2 | hex_encode, toupper, startswith, replace | ~30 |

## 六、Flow 解释器

```java
@Component
public class FlowInterpreter {
    private static final Pattern STEP_REF = Pattern.compile("http\\((\\d+)\\)");

    /**
     * null / 空 → 顺序执行全部步骤
     * "http(1) && http(2)" → AND 逻辑
     * "http(1) || http(2)" → OR 逻辑
     */
    public boolean execute(List<HttpStep> steps, String flow, StepExecutor executor) {
        if (flow == null || flow.isBlank()) return sequential(steps, executor);
        // 解析 && / || 表达式 → 执行 → 逻辑运算
    }
}
```

## 七、HttpExecutor — 协议执行

```
HttpExecutor
  ├─ HttpClient (连接池复用, HTTP/1.1 & HTTP/2)
  ├─ execute(StepDef, vars) → normal 模式（method + path + headers + body）
  ├─ executeRaw(String raw, vars) → 原始 HTTP 文本解析
  ├─ 重定向：follow + max-redirects + host-redirects
  ├─ CookieJar：cookie-reuse 支持
  └─ 超时：连接 3s，请求 15s
```

## 八、ResultWriter — 批量写入

```java
@Component
public class ResultWriter {
    private final Queue<DetectionResult> buffer = new ConcurrentLinkedQueue<>();
    private final DetectionResultMapper mapper;

    public void write(DetectionResult result) {
        buffer.add(result);
        redisIncr(result);                // Redis INCR task:{taskId}:{status}
        if (buffer.size() >= 500) flush();
    }

    @Scheduled(fixedDelay = 5000)
    public void flushByTimeout() { flush(); }

    private void flush() {
        List<DetectionResult> batch = drain();
        if (!batch.isEmpty()) mapper.insert(batch);
    }
}
```

去 `synchronized`，用 `ConcurrentLinkedQueue`。去循环单条 insert，用 MyBatis-Plus `saveBatch`。

## 九、DetectionEngine — 纯编排

```java
@Component
public class DetectionEngine {
    private final HttpExecutor httpExecutor;
    private final MatcherPipeline matcherPipeline;
    private final ExtractorPipeline extractorPipeline;
    private final FlowInterpreter flowInterpreter;
    private final ResultWriter resultWriter;

    public void execute(TaskItemMessage msg) {
        DetectionResult result = initResult(msg);
        try {
            VariableContext vars = VariableContext.from(msg);
            boolean matched = flowInterpreter.execute(msg.getSteps(), msg.getFlow(),
                step -> executeStep(step, vars));
            result.setStatus(matched ? "matched" : "not_matched");
        } catch (HttpException e) {
            result.setStatus("error"); result.setErrorMessage(e.getMessage());
        } catch (Exception e) {
            log.error("检测异常 taskId={}", msg.getTaskId(), e);
            result.setStatus("error"); result.setErrorMessage("内部错误");
        }
        resultWriter.write(result);
    }

    private boolean executeStep(HttpStep step, VariableContext vars) {
        HttpResponseContext ctx = httpExecutor.execute(step, vars);
        vars.updateFrom(ctx);
        extractorPipeline.extract(ctx, step.getExtractors(), vars);
        return step.hasMatchers()
            ? matcherPipeline.evaluate(ctx, step.getMatchers(), step.getMatchersCondition())
            : false;
    }
}
```

## 十一、组件清单

| 组件 | 当前 | 3.0 |
|------|------|-----|
| `MatcherChain` | 硬编码 3 种 + switch 取 rule | `MatcherPipeline` + `MatcherRegistry`（Spring 自动发现） |
| `ExtractorChain` | if-else 分发 2 种 | `ExtractorPipeline` + `ExtractorRegistry` |
| Matcher 策略 | `new` 手动构造，非 Bean | `@Component`，可注入依赖 |
| RegexMatcher | **缺失** | 新增 |
| DSL 引擎 | MVEL + preprocess 扭曲 | Aviator（沙箱安全） |
| `DetectionEngine` | 150 行含 DTO 映射 | ~50 行纯编排 |
| `FlowInterpreter` | 嵌在 Engine 中 | 独立组件 |
| `HttpExecutor` | 无连接池/重试/Cookie | 连接池 + 重定向 + CookieJar |
| `ResultWriter` | synchronized + 循环单条 insert | `ConcurrentLinkedQueue` + `saveBatch` |
| `templateId` | `yamlId.hashCode()` | `templateDbId`（DB 主键） |
| `DetectionResult.id` | 实体字段名 | `resultId`（对齐 DDL） |
