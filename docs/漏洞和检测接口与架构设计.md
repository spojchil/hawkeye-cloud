# 漏洞管理服务 & 检测服务 — 接口与架构设计 v2.0

> 基于新 10 表 schema。vul-service 是知识库，detection-service 是执行引擎。

---

## 一、整体交互

```
                    ┌─────────────┐
                    │   Gateway   │
                    └──────┬──────┘
              ┌────────────┼────────────┐
              ▼            ▼            ▼
        ┌─────────┐  ┌─────────┐  ┌──────────┐
        │  Vul    │  │  Task   │  │ Detection│
        │ :8004   │  │ :8005   │  │ :8006+   │
        └────┬────┘  └────┬────┘  └────┬─────┘
             │ Feign       │ RocketMQ   │ Feign
             └─────────────┴────────────┘
                        共享 MySQL
```

**调用关系：**
- 前端 → Gateway → vul-service（模板 CRUD）
- task-service → RocketMQ → detection-service（下发任务）
- detection-service → Feign → vul-service（获取模板检测配置）
- detection-service → Feign → asset-service（获取资产信息）
- detection-service → 写 detection_result 表

## 五、Service 层设计

### 5.1 Vul-Service

```
VulTemplateService
  ├── pageQuery(VulTemplatePageRequest) → IPage<VulTemplatePageResponse>
  ├── getDetail(Long id) → VulTemplateDetailResponse
  ├── create(VulTemplateRequest) → Long
  ├── update(Long id, VulTemplateRequest) → void
  ├── delete(Long id) → void
  ├── batchDelete(List<Long> ids) → int
  ├── setEnabled(Long id, boolean enabled) → void
  └── getForDetection(Long id) → VulTemplateDetectDTO       // Feign用

VulCategoryService
  ├── tree(Long parentId) → List<VulCategoryResponse>
  ├── create(VulCategoryRequest) → Long
  ├── update(Long id, VulCategoryRequest) → void
  ├── delete(Long id) → void
  ├── addTemplates(Long categoryId, List<Long> templateIds) → int
  ├── removeTemplates(Long categoryId, List<Long> templateIds) → int
  └── listTemplates(Long categoryId, PageQuery) → IPage<VulTemplatePageResponse>

VulTagService
  └── list(String keyword) → List<VulTagResponse>

VulTemplateAssemblyService  (组装详情DTO)
  ├── assembleDetail(VulTemplate entity) → VulTemplateDetailResponse
  └── assembleDetectDTO(VulTemplate entity) → VulTemplateDetectDTO
```

组装策略（给详情/Feign用）：
- 分步并行查询：template → tags(1次) + references(1次) + httpSteps(1次) + matchers(1次) + extractors(1次) + categories(1次)
- 共约 6-7 次简单查询，避免复杂 JOIN
- 每个步骤查询都可被缓存

### 5.2 Detection-Service

```
DetectionEngine  (核心编排)
  └── execute(TaskItemMessage msg) → DetectionResult

TemplateFetcher  (缓存穿透)
  ├── fetch(Long vulId) → VulTemplateDetectDTO
  └── 缓存: Caffeine L1 (2000条/30min) + Redis L2 (30min)

VariableResolver  (占位符替换)
  ├── buildContext(Asset asset, VulTemplateDetectDTO template) → VariableContext
  └── resolve(String template, VariableContext) → String

FlowExecutor  (多步骤解析)
  └── buildPlan(VulTemplateDetectDTO) → ExecutionPlan

HttpExecutor  (HTTP 请求)
  ├── execute(HttpRequestConfig config) → HttpResponseContext
  └── 策略: SimpleRequest / RawRequest

MatcherChain  (匹配判定)
  ├── match(HttpResponseContext ctx, List<MatcherConfig> matchers, String condition) → MatchResult
  └── 实现: WordMatcher / StatusMatcher / DslMatcher / RegexMatcher / SizeMatcher

ExtractorChain  (变量提取)
  └── extract(HttpResponseContext ctx, List<ExtractorConfig> extractors) → Map<String, String>

ResultWriter  (批量写入)
  ├── onResult(DetectionResult result)
  ├── 本地缓冲 500条 / 5s定时 → 批量 INSERT
  └── Redis INCR task:{taskId}:completed / failed

DetectionResultService  (结果查询)
  ├── pageQuery(DetectionResultPageRequest) → IPage<DetectionResultResponse>
  ├── getById(Long id) → DetectionResultResponse
  ├── stats(Long taskId) → DetectionStatsResponse
  └── verify(DetectionVerifyRequest) → DetectionVerifyResponse
```

## 六、检测引擎执行流程

```
Consumer 收到 TaskItemMessage {taskId, itemId, assetId, vulId}
  │
  ├─ 1. TemplateFetcher.fetch(vulId)
  │     └─ Caffeine L1 → Redis L2 → Feign vul-service
  │
  ├─ 2. AssetFetcher.fetch(assetId)
  │     └─ Feign asset-service → Asset{url, protocol, host, port}
  │
  ├─ 3. VariableResolver.buildContext(asset, template)
  │     └─ {{BaseURL}} → https://example.com
  │     └─ {{Hostname}} → example.com
  │     └─ template.variables → ctx
  │
  ├─ 4. FlowExecutor.buildPlan(template)
  │     └─ null flow → 单步计划
  │     └─ "http(1) && http(2)" → 两步串行计划
  │
  ├─ 5. for each step in plan:
  │     ├─ VariableResolver.resolve(step.config) → 替换占位符
  │     ├─ HttpExecutor.execute(config) → HttpResponseContext
  │     ├─ ExtractorChain.extract(response, step.extractors) → 变量写回ctx
  │     └─ MatcherChain.match(response, step.matchers, step.matchersCondition)
  │
  ├─ 6. 汇总最终结果 → DetectionResult
  │
  └─ 7. ResultWriter.onResult(result)
        └─ buffer.add → size≥500? → batch INSERT
        └─ Redis INCR task:{taskId}:completed
```

## 七、实施优先级

| 优先级 | 服务 | 模块 | 内容 |
|--------|------|------|------|
| P0 | vul | Entity 层 | 10 个新 Entity + 修复 VulSeverityEnum |
