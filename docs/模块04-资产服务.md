# 资产服务（asset-service）

**端口：** `:8003`
**状态：** 已完成

---

## 定位

资产（目标 URL/域名）的管理、分类、查询，是漏洞检测的数据基础。

## 子模块结构

| 子模块 | 关键类 | 职责 |
|--------|--------|------|
| asset-common | `Asset`、`AssetCategory`、`AssetCategoryMapping`（实体）、`BaseAssetEntity`、枚举、VO | 实体、DTO、枚举 |
| asset-api | `AssetController`、`AssetCategoryController` | REST API |
| asset-business | `AssetService`、`AssetServiceImpl`、`AssetCategoryService`、Mapper、MapStruct | 业务逻辑 |
| asset-bootstrap | `AssetApplication` | 启动类 |

## 数据模型

| 表 | 说明 | 主要字段 |
|----|------|----------|
| `asset` | 资产主表 | name, url, protocol, host, port, risk_level, status |
| `asset_category` | 资产分类（树形） | name, parent_id, description, sort_order |
| `asset_category_mapping` | 资产-分类关联 | asset_id, category_id |

## API 端点

### 资产接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/asset` | 分页查询，支持 name/host/risk/status/category 多条件过滤 |
| POST | `/asset` | 创建资产 |
| GET | `/asset/{id}` | 获取资产详情 |
| PUT | `/asset/{id}` | 更新资产 |
| DELETE | `/asset/{id}` | 删除资产（有分类关联时拒绝删除） |

### 分类接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/asset-category` | 查询分类列表（按 parentId + name 筛选） |
| POST | `/asset-category` | 创建分类（不传 parentId = 顶级分类） |
| PUT | `/asset-category/{id}` | 更新分类（变更父级时防环校验） |
| DELETE | `/asset-category/{id}` | 删除分类（有子分类/关联资产时拒绝） |
| POST | `/asset-category/{id}/assets` | 批量添加资产到分类 |
| DELETE | `/asset-category/{id}/assets` | 从分类批量移除资产 |

## 设计要点

1. **分类树** — `parent_id IS NULL` 表达顶级分类，前端根据平铺列表自行构建树
2. **删除保护** — 级联检查：子分类 + 关联资产
3. **防环检查** — 更新 parentId 时校验目标父节点不能是当前节点的子孙
4. **多对多** — 资产与分类通过中间表关联

## 枚举

- `AssetStatusEnum` — DISABLED / ENABLED / DEPRECATED
- `AssetRiskEnum` — UNKNOWN / LOW / MEDIUM / HIGH
