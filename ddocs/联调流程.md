# 联调流程图

> 从 "提交任务" 到 "查看结果" 的完整数据流。

---

## 一、全链路时序

```mermaid
sequenceDiagram
    participant FE as 前端/curl
    participant GW as Gateway :8001
    participant Task as task-service :8005
    participant Vul as vul-service :8004
    participant Asset as asset-service :8003
    participant RMQ as RocketMQ :9876
    participant Detect as detection-service :8006
    participant Redis as Redis :6379
    participant DB as MySQL :3306

    FE->>GW: POST /api/task {assetIds, templateIds}
    GW->>Task: 转发

    Task->>DB: INSERT task (待执行)
    Task-->>FE: {taskId}  ← 立刻返回

    Note over Task: ── 异步线程 ──

    Task->>Vul: Feign GET /vul/internal/{id}  ×N
    Vul-->>Task: VulTemplateDetectDTO
    Task->>Asset: Feign GET /asset/{id}  ×N
    Asset-->>Task: Asset

    Task->>DB: INSERT task_item (M×N条)
    Task->>Task: 构建 TaskItemMessage(含全量数据)

    loop M×N 条
        Task->>RMQ: 异步发送 task_item_topic
    end

    Note over Detect: ── 消费执行 ──

    RMQ->>Detect: Push TaskItemMessage
    Detect->>Detect: VariableResolver(递归解析)
    Detect->>Detect: HttpExecutor(GET/POST)
    Detect->>Detect: MatcherChain(Word/Status/Dsl)
    Detect->>Redis: INCR task:{taskId}:completed
    Detect->>DB: 批量 INSERT detection_result

    Note over Task: ── 进度感知 ──

    loop 每 2s
        Task->>Redis: GET task:{taskId}:completed
    end
    completed >= total → Task->>DB: UPDATE task.status = 已完成

    FE->>GW: GET /api/task/{id}
    GW->>Task: 转发
    Task->>DB: SELECT task
    Task-->>FE: {status, totalItems, completedItems...}
```

## 二、服务端口与中间件

| 服务/中间件 | 端口 | 用途 |
|------------|------|------|
| Gateway | 8001 (外部 8000) | 路由 + JWT 校验 |
| auth-service | 8002 | 登录认证 |
| asset-service | 8003 | 资产 CRUD |
| vul-service | 8004 | 模板 CRUD + 检测配置 |
| task-service | 8005 | 任务调度 |
| detection-service | 8006 | HTTP 探测执行 |
| MySQL | 3306 | 数据存储 |
| Redis | 6379 | 进度计数 + 缓存 |
| RocketMQ | 9876 | 异步消息 |
| Nacos | 8848 | 服务发现 |

## 三、联调步骤

### 前置准备
```bash
# 1. 确保所有中间件运行
# MySQL / Redis / RocketMQ / Nacos

# 2. 确保 6 个服务全部启动，Nacos 注册中心可见

# 3. 创建 RocketMQ Topic（如未自动创建）
mqadmin updateTopic -n localhost:9876 -t task_item_topic -c DefaultCluster
```

### 验证链路
```bash
# Step 1: 获取 token（若通过 Gateway 测试）
curl -X POST http://localhost:8000/api/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"username":"admin","password":"123456"}'

# Step 2: 查模板 ID（记几个存在的 ID）
curl http://localhost:8004/vul?size=3

# Step 3: 查可用资产
curl http://localhost:8003/asset

# Step 4: 创建任务
curl -X POST http://localhost:8005/task \
  -H 'Content-Type: application/json' \
  -d '{"taskName":"联调","assetIds":[1],"templateIds":[1001],"priority":1}'

# Step 5: 查进度
curl http://localhost:8005/task/1

# Step 6: 查结果（直接查 DB）
mysql -u root -p hawkeye -e "SELECT * FROM detection_result WHERE task_id=1"
```

## 五、常见问题速查

| 现象 | 原因 | 检查 |
|------|------|------|
| task 状态"异常终止" | 模板/资产 Feign 调用失败 | task-service 控制台看"任务拆分异常"日志 |
| task 一直"执行中" | detection 没消费或 Redis 不通 | detection 控制台看"收到检测消息" |
| 结果里有 error | 模板 URL 含特殊字符 | 看 detection 日志的IllegalArgumentException |
| Gateway 返回 401 | 没传 JWT token | 先调 /auth/login 拿 token |
| Feign 调用 500 | DB 表结构和 Entity 不匹配 | 比对 vul-ddl.sql 和实体 |
