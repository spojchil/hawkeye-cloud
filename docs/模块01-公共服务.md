# 公共服务（common-service）

**路径：** `common-service/common-utils/`

**定位：** 所有微服务的公共依赖，提供通用能力和基础设施，不独立部署。

---

## 核心组件

| 功能 | 类/文件 | 说明 |
|------|---------|------|
| 统一响应 | `ApiResponse<T>` | 全局统一 JSON 响应格式，包含 code、message、data |
| 业务异常 | `ApiException` | 运行时异常，携带 ErrorCode |
| 错误码 | `ErrorCode` | 错误码枚举接口 |
| 分页模型 | `ListResult<T>` | 封装分页查询结果（records、total、page、size） |
| 请求上下文 | `RequestContext` | ThreadLocal 存储当前请求的租户 ID |
| 请求上下文 Filter | `RequestContextFilter` | Servlet Filter，初始化/清理 RequestContext |
| 请求头常量 | `HeaderConstants` | X-ACCOUNT-ID、X-TENANT-ID 等 |
| MyBatis-Plus 配置 | `MybatisPlusConfig` | 多租户拦截器 + 分页拦截器 |
| 自动填充 | `MybatisMetaObjectHandler` | 自动填充 create_time、update_time、deleted_at |
| 多租户拦截器 | `MultiTenantInterceptor` | TenantLineHandler，SQL 自动注入 tenant_id |
| 日志切面 | `@LogExecutionTime` + `LogExecutionTimeAspect` | 方法执行耗时统计 |
| 全局异常处理 | `GlobalExceptionHandler` | 统一异常拦截与响应 |

---

## 公共实体基类 `BaseEntity`

```java
public class BaseEntity {
    private Long id;
    private Long tenantId;
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private Long deletedAt;    // 时间戳软删除
    private String createBy;
    private String updateBy;
}
```

所有业务实体继承此基类，获得统一的 id、租户隔离、审计字段。

---

## 多租户机制

基于 MyBatis-Plus 的 `TenantLineHandler` 实现，对业务代码零侵入：

1. 请求经 `RequestContextFilter` 从 Header 提取 `X-TENANT-ID`
2. 存入 `RequestContext`（ThreadLocal）
3. `MultiTenantInterceptor` 在 SQL 执行时自动拼接 `AND tenant_id = ?`
4. 请求结束后 Filter 清理 ThreadLocal，防止内存泄漏

支持按表粒度控制是否需要多租户过滤（如 account 表跳过过滤）。
