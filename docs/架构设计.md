## 整体架构

```mermaid
graph TD
    Browser[浏览器 / 客户端] --> Nginx[Nginx<br/>反向代理 + 负载均衡]
    Nginx --> GW

    subgraph Gateway ["Gateway :8001"]
        GW[路由转发 + CORS]
        GW --> AuthFilter{AuthFilter}
    end

    AuthFilter -->|白名单 /api/auth/**| Auth
    AuthFilter -->|非白名单<br/>JWT 校验 + 黑名单| Route[路由转发<br/>注入 X-ACCOUNT-ID / X-TENANT-ID]

    subgraph 基础服务
        Auth[Auth :8002<br/>认证服务]
    end

    subgraph 核心服务
        Asset[Asset :8003<br/>资产管理]
        Vul[Vul :8004<br/>漏洞模板管理]
        Task[Task :8005<br/>任务调度引擎]
    end

    Route --> Asset
    Route --> Task

    Task -->|RocketMQ 工作队列| Detection

    subgraph Detection ["Detection :8006+"]
        Worker1[Worker 1]
        Worker2[Worker 2]
        WorkerN[Worker N]
    end

    Detection -->|更新计数| Redis[(Redis)]
    Task -.->|轮询进度| Redis
    Task -.->|获取资产| Asset
    Task -.->|获取模板| Vul
```

### 服务职责矩阵

| 服务 | 端口 | 类别 | 职责 | 依赖 |
|------|------|------|------|------|

| auth-service | 8002 | 基础 | 用户认证、Token 签发 | MySQL、Redis |
| asset-service | 8003 | 核心 | 资产 CRUD、分类管理 | MySQL |

## 分层架构（每个微服务内部）

```mermaid
graph TD
    subgraph 服务边界
        Bootstrap[bootstrap<br/>启动层<br/>启动类 + application.yml]
        API[api<br/>Controller 层<br/>REST 端点 + @Valid 校验]
        Business[business<br/>Service + Mapper 层<br/>业务逻辑 + 数据访问]
        Common[common<br/>实体、配置、工具层<br/>Entity / VO / Enum / Config]
    end

    Bootstrap --> |依赖| API
    API --> |依赖| Business
    Business --> |依赖| Common
```

这种分层的优势：
- **依赖单向**：bootstrap → api → business → common，无循环依赖
- **职责清晰**：启动、接口暴露、业务实现、公共基础设施各司其职
- **便于测试**：business 层的 Service 可独立单元测试

## 网关设计

网关使用 Spring Cloud Gateway，配置路由规则、CORS 和全局 JWT 认证过滤器：

### 认证链路

```mermaid
graph TD
    Client[客户端请求] --> GW[Gateway :8001]

    GW --> Filter{AuthFilter<br/>全局认证过滤器}

    Filter -->|白名单路径<br/>/api/auth/**| Pass[直接放行]
    Filter -->|非白名单路径| JWT[提取 JWT<br/>校验签名 + 有效期]
    JWT --> Blacklist{Redis 黑名单?}
    Blacklist -->|在黑名单中| Reject[拒绝请求]
    Blacklist -->|不在黑名单| Inject[注入 Header<br/>X-ACCOUNT-ID<br/>X-TENANT-ID]

    Pass --> Route[路由转发到下游服务]
    Inject --> Route
```

## 多租户设计

基于 MyBatis-Plus 的 `TenantLineHandler` 实现：

1. 请求通过 `RequestContextFilter` 拦截，从 Header 提取 `X-TENANT-ID`
2. 存入 `RequestContext`（ThreadLocal）
3. `MultiTenantInterceptor`（TenantLineHandler）在 SQL 执行时自动拼接 `AND tenant_id = ?`
4. `RequestContext` 在请求结束后由 Filter 清理，防止内存泄漏

## 检测链路（核心）

核心检测流程由四个核心服务协作完成：

```mermaid
graph LR
    Client[客户端] -->|POST /task| TaskAPI

    subgraph Task ["task-service :8005"]
        TaskAPI[同步: 持久化<br/>立刻返回 taskId]
        TaskAPI -.-> ThreadPool[异步线程池]
        Validate[校验资产 + 模板]
        Split[拆分 资产×漏洞]
        Dispatch[异步投递]
        Validate --> Split --> Dispatch
    end

    Dispatch -->|RocketMQ| RMQ{{RocketMQ<br/>taskId + itemId}}

    subgraph Detection ["detection-service :8006+"]
        Consumer[消费 TaskItem]
        Engine[DetectionEngine<br/>FlowInterpreter 编排]
        Http[HttpExecutor]
        Match[MatcherPipeline<br/>Word/Status/Dsl/Regex]
        Extract[ExtractorPipeline<br/>Regex/Kval]
        Writer[ResultWriter<br/>批量写 DB + Redis 计数]
        Consumer --> Engine
        Engine --> Http
        Http --> Match
        Http --> Extract
        Engine --> Writer
    end

    RMQ --> Consumer
    
    Writer --> Redis[(Redis<br/>completed/failed/remaining)]
    ThreadPool -.->|定时轮询进度| Redis

    Consumer -.->|重试耗尽| DLQ[%DLQ% 死信队列]
```

1. **用户提交任务** → `task-service` 同步持久化，立刻返回 `taskId`
2. **异步校验拆分** → 内部线程池校验资产 URL + 漏洞模板完整性，拆分为 `资产 × 漏洞` 检测项
3. **异步投递** → 逐条异步发送到 RocketMQ，由消费端集群竞争消费
4. **Worker 并发执行** → `detection-service` 消费消息，VariableContext 解析变量 → HttpExecutor 探测 → MatcherPipeline 匹配 → ExtractorPipeline 提取 → FlowInterpreter 执行流编排
5. **结果写入** → ResultWriter 本地缓冲后批量写 DB + Redis INCR 计数
6. **进度感知** → `task-service` 定时轮询 Redis 检查 `completed >= total`，判定任务完成
7. **失败处理** → 重试耗尽后自动进入 RocketMQ 原生死信队列

## 规划中服务

| 服务 | 说明 | 优先级 |
|------|------|--------|
| tenant-service | 多租户管理（SaaS 隔离）：租户 CRUD、配额管理 | P1 |
| audit-service | 日志审计服务：操作记录、安全审计、合规 | P2 |

当前多租户通过 MyBatis-Plus `TenantLineHandler` 在 SQL 层面实现，租户 ID 由网关注入 Header。独立租户服务将提供更完整的管理能力。
